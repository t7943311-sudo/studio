NovaAdmin React Template

Your guide to getting started

Introduction

Thank you for choosing NovaAdmin! This is a professional, frontend-only React admin dashboard template built with a modern technology stack. It's designed to be clean, customizable, and easy for developers of all skill levels to get started with.

This template is built with Next.js (App Router), Tailwind CSS, and shadcn/ui, providing a powerful and efficient development experience.

Project Setup

To get the project running on your local machine, please follow these simple steps. You will need Node.js (v18 or newer) and npm installed.

  1. Unzip the Package:

    First, unzip the downloaded file from your purchase.

  2. Open Terminal:

    Navigate to the project's root folder (the one containing package.json) in your terminal or code editor.

  3. Install Dependencies:

    Run the following command to install all the required libraries for the project.

    npm install
  4. Start the Development Server:

    Once the installation is complete, start the local development server with this command.

    npm run dev

Your browser should automatically open to http://localhost:9002 with the application running. You are now ready to start customizing!

Folder Structure

The project uses the standard Next.js App Router structure, which is organized for scalability and clarity. Here's an overview of the key folders:

/src
├── app/
│   ├── (main)/             // Main layout for the dashboard pages
│   │   ├── dashboard/
│   │   ├── users/
│   │   ├── ui-elements/      // Page showcasing all UI components
│   │   └── ... (other pages)
│   ├── login/              // Login page (has a separate, simpler layout)
│   ├── layout.tsx          // Root layout of the entire application
│   └── globals.css         // Global styles and theme color variables
│
├── components/
│   ├── layout/             // Main layout components (Header, Sidebar, etc.)
│   ├── ui/                 // Reusable UI components from shadcn/ui
│   └── theme-provider.tsx  // Logic for switching between light/dark mode
│
├── contexts/
│   └── sidebar-provider.tsx // State management for the collapsible sidebar
│
├── lib/
│   ├── mock-data.ts        // All placeholder data for tables and charts
│   └── utils.ts            // Utility functions
│
└── public/                 // Static assets like images (if any)

Customization

  • Changing the Logo:

    The logo is an SVG component located at /src/components/logo.tsx. You can edit this file to insert your own SVG code. Alternatively, you can modify its usage in /src/app/(main)/layout.tsx and /src/components/layout/mobile-sidebar.tsx to use an <Image> tag from next/image if you prefer using a PNG or JPG file placed in the /public folder.

  • Changing Colors & Theme:

    All theme colors are defined as CSS variables in /src/app/globals.css. You can change the HSL values for primary color, background, text, etc., in the :root (for light mode) and .dark (for dark mode) selectors. This makes it easy to re-brand the entire template.

    :root {
    --primary: 243 75% 59%; /* This is the main theme color (#4F46E5) */
    --background: 210 40% 98%;
    /* ... other colors */
}
  • Adding a New Page:

    To add a new page that uses the main dashboard layout (with sidebar and header):

    1. Create a new folder inside /src/app/(main)/. For example, /src/app/(main)/mynewpage/.
    2. Inside this new folder, create a file named page.tsx.
    3. Add your page content as a React component in this file. For example: export default function MyNewPage() { return <h1>Hello World</h1>; }.
    4. Finally, add a link to your new page in the sidebar navigation file: /src/components/layout/sidebar-nav.tsx. Just add a new object to the navItems array.

Routing

This template uses the Next.js App Router. Routing is file-system based, which makes it intuitive.

  • A folder inside /app creates a new URL segment. For example, a file at /app/users/page.tsx corresponds to the /users URL.
  • The (main) folder is a "Route Group". It's a special Next.js feature that organizes files without affecting the URL. This allows all pages within it to share the main dashboard layout defined in /app/(main)/layout.tsx.
  • Pages like Login, Register, and 404 are outside the (main) group, so they have a different, simpler layout without the sidebar and header.
